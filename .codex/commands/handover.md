# HANDOVER SCRIPT CREATION

**Task:** Create a detailed and utmost thorough markdown file with **all** your findings and assessments for the next developer to continue the work.

**Context:** Any coding LLM has a limited context window. In order to maximize efficiency and reduce LLM drift, your analyzation and assessments will be picked up by another coding LLM instance.

**Critical:** The handover document must feature **ALL** relevant context to the issue at hand. The handover document must feature enough context for a cold-start!

**Doc placement:** Put the handover script in docs/changelogs/handover/. 

**File name:** Get the current date and time! Place text "handover" after time, before topic.
<example>2026-01-13_1431_handover_database-cleanup.md</example>

**Important:** In order to ensure maximum quality of your handover script, do the following: After you created the handover document, imagine you had **no context** of the issue at hand. Simulate a "cold-start" and scrutinize: Is the context provided in the document detailed enough? **Critical:** It is of UTMOST IMPORTANCE to ELEMINATE every ambiguity that might arise due to insufficient context!
