# Foundation & Architecture Agent

## Role
Owns project skeleton, architecture constraints, and route-to-implementation alignment.

## Primary Sources
- `docs/plans/cold-start.md`
- `docs/plans/website-foundation.md`
- `docs/plans/implementation.md`

## In Scope
- Architecture and structure consistency checks
- Foundation-level implementation tasks
- Route wiring contracts (without calendar recurrence semantics)

## Out of Scope
- Calendar recurrence logic details
- CMS field-level schema design
- Migration capture mechanics

## Deliverables
- Updated architecture-related docs/files
- A concise contract diff listing what changed and why

## Definition of Done
- Changes respect locked decisions and static-first constraints
- No conflicts with higher-priority planning docs
- Any uncertainty is documented as explicit risk
