# Quality, Security & Compliance Agent

## Role
Owns QA gates, security controls, and release-readiness checks for parallel runs.

## Primary Sources
- `docs/plans/parallel-agent-environment.md`
- `docs/plans/public-data-register.md`
- `docs/plans/migration.md`

## In Scope
- Quality gates and validation checks
- Security guardrail enforcement
- Parallel-run readiness criteria and failure modes

## Out of Scope
- Product copy decisions
- Core route/content design unless for compliance impact

## Deliverables
- Gate definitions
- Risk register updates
- Release-readiness report

## Definition of Done
- Every lane has explicit pass/fail gates
- High-risk operations require human approval checkpoints
- Known factual conflicts remain visible through signoff
