# 2026-04-10 1907 phase 6 closeout

## Scope

This changelog records the Phase 6 closeout session that followed the earlier migration batch.

It covers:

- retained-evidence parity cleanup for homepage, about, and membership content
- event migration reconciliation against the retained events index and detail evidence
- manifest status normalization from blanket `qa-pending` to concrete `migrated` or `blocked`
- control-doc updates that move the active phase from Phase 6 to Phase 7

## Why this work was needed

- the previous Phase 6 batch left homepage and static-page copy only partially audited against the retained local HTML
- the representative `Sunday Funday` event detail existed, but the retained events index did not support treating it as an evidence-backed weekly recurring series
- `migration/manifest.json` still used `qa-pending` for records that had already been migrated or should have been explicitly blocked

## What changed

### Event reconciliation

- updated `src/content/events/sunday-funday.md`

The retained `/veranstaltungen/` index shows no active April 2026 entries, so `Sunday Funday` was reconciled to the single retained detail occurrence on `2025-11-30`.

The hidden seed-only event placeholder was removed so the content collection reflects real migrated evidence instead of migration scaffolding.

### Migration tracking and control docs

- updated `migration/manifest.json`
- updated `README.md`
- updated `docs/plans/execution-plan.md`
- updated `docs/plans/status-checklist.md`

The migration manifest now records touched records as `migrated` or `blocked` instead of leaving them under a generic `qa-pending` umbrella.

`parkausbesserungen` now stays explicitly `blocked` because the retained evidence remains truncated and fuller body recovery would require better local evidence.

The audited control state now marks Phase 6 complete and makes Phase 7 the active continuation phase.
